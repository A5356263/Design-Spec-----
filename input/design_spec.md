# 页面需求：差旅报销申请创建页

## 页面目标
创建一个企业内部使用的差旅报销申请页面。
页面重点是：填写、检查、提交。
这是一个创建页，不是列表页，不是审批详情页。

## 页面整体布局
页面采用三段结构：

1. 顶部标题与操作区
2. 中间主体区，采用左右布局
3. 底部提交操作区

## 顶部标题与操作区
顶部需要包含：

- 页面标题：差旅报销申请
- 页面说明：用于创建并提交本次差旅报销
- 状态标签：草稿
- 操作按钮：
  - 保存草稿
  - 提交审批

## 中间主体区
中间主体区采用左右布局：

- 左侧为主要填写区，占主要宽度
- 右侧为辅助信息区，占较小宽度

## 左侧主要填写区

### 模块一：基础信息
使用表单分组展示，字段固定为：

- 报销单编号（只读，占位展示即可）
- 申请人
- 所属部门
- 出差事由
- 出差开始日期
- 出差结束日期

### 模块二：费用明细
使用表格展示费用明细。

表格列固定为：

- 日期
- 费用类型
- 金额
- 说明

请先给出 2 条示例数据：

1. 2026-03-01 / 酒店 / ¥800 / 上海出差住宿
2. 2026-03-02 / 餐饮 / ¥120 / 客户拜访午餐

### 模块三：补充说明
使用多行输入区域，标题为：补充说明

## 右侧辅助信息区

### 卡片一：报销规则提示
展示以下静态提示：

- 酒店费用需提供发票
- 餐饮费用单次不可超过规定额度
- 提交后将进入审批流程

### 卡片二：汇总信息
展示以下静态汇总：

- 费用合计：¥920
- 明细条数：2
- 当前状态：草稿

## 底部操作区
底部固定展示两个按钮：

- 取消
- 提交审批

底部操作区应与主体内容有清晰间距。

## 风格要求
- 标准企业后台风格
- 分组清楚
- 层级清楚
- 不追求复杂视觉
- 优先使用标准组件
- 不自造复杂组件

## 三阶段要求

### 阶段一：骨架阶段
只生成：
- 顶部区
- 左右主体区
- 底部区
- 各模块容器
- 主次布局关系

不要生成：
- 真实表格数据
- 真实提示文案
- 复杂状态内容

### 阶段二：组件阶段
补充：
- 表单结构
- 表格结构
- 按钮
- 提示卡片
- 状态标签
- 汇总卡片

不要重做骨架布局。

### 阶段三：内容阶段
补充：
- 示例文案
- 表格示例数据
- 汇总数值
- 状态信息

不要修改组件类型与大区块结构。
